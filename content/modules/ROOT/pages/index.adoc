Welcome to the *Defend, Contain, Comply* workshop.

You are a platform engineer at *{company_name}*. A Splunk alert fires: *{target_cve}* -- a critical Apache httpd SSRF vulnerability (CVSS 9.8) -- has been detected on your customer-facing application server.

No patch exists yet. You must *defend* the system, *contain* the threat when a patch arrives, and *comply* by delivering a hardened container image.

== What You Will Learn

* **Module 1 -- DEFEND**: Detect the CVE, contain the vulnerable service, and harden the system posture using Event-Driven Ansible responding to Splunk alerts.
* **Module 2 -- CONTAIN**: Apply advisory *{target_advisory}* through a policy-gated workflow with approval nodes, pre-patch validation, and compliance reporting.
* **Module 3 -- COMPLY**: Build a hardened container image from the patched system, scan it, and push it to a local registry.
