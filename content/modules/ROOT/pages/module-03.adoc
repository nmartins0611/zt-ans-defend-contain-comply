== Objective

Build a hardened container image from the patched application server, scan it for vulnerabilities, and push it to a local registry -- establishing a compliant container supply chain.

== Scenario

With {target_cve} patched and the system hardened, you need to deliver the application as an immutable container image. The image must be built from a trusted base (UBI9 minimal), include the patched `httpd`, run as a non-root user, and pass a vulnerability scan before being pushed to the registry.

== Exercises

=== Exercise 3.1: Build the Container Image

Run the *Build Container* job template. This uses Podman to build a container image from a templated Containerfile based on `ubi9/ubi-minimal` with the patched `httpd` installed.

=== Exercise 3.2: Scan the Image

Run the *Scan Image* job template. This scans the built image for known vulnerabilities and verifies that {target_cve} is not present.

=== Exercise 3.3: Push to Registry

Run the *Push to Registry* job template. This pushes the verified image to the local Podman registry running on the app server.

Verify the image is accessible from the registry.

== Summary

You have completed the full vulnerability lifecycle: from detection and containment, through policy-gated patching, to delivering a hardened container image. The application is now running from an immutable, scanned, and registered container.
