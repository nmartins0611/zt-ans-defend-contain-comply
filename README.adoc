= Defend, Contain, Comply
:toc: auto

A hands-on workshop demonstrating automated vulnerability response, policy-gated
patching, and container supply-chain compliance using Red Hat Ansible Automation
Platform.

== Workshop Modules

[cols="1,3"]
|===
| Module | Focus

| *DEFEND*
| Detect CVE-2024-38476 in Apache httpd, contain the vulnerable service, and harden the system posture -- all driven by Event-Driven Ansible responding to Splunk alerts.

| *CONTAIN*
| Apply the RHSA-2024:5138 patch through a policy-gated workflow with pre-patch validation, approval gates, post-patch verification, and compliance reporting.

| *COMPLY*
| Build a hardened container image from the patched system, scan it for vulnerabilities, and push it to a local registry -- establishing a compliant supply chain.
|===

== Architecture

[cols="1,2"]
|===
| Host | Role

| `control`  | AAP Controller 2.6 + EDA Controller
| `central`  | Splunk (SIEM), OPA (policy engine), errata yum repo
| `rhel01`   | Target app server (Meridian Financial customer portal)
| `rhel02`   | Spare RHEL host
| `vault`    | HashiCorp Vault (credential management)
| `gitea`    | Git repository (containerized)
|===

== Repository Structure

* `config/` -- VM topology, firewall rules, secrets
* `content/` -- Antora lab guide (AsciiDoc)
* `setup-automation/` -- Provisioning playbooks and bootstrap scripts
* `runtime-automation/` -- Module transition scripts (progressive content reveal)

== Content Repository

== Prerequisites

* AAP 2.6 controller image
* RHEL 9.5 base image
